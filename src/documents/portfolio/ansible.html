ANSIBLE

IF YOUR OPERATIONAL PARADIGM IS TO BURN DOWN AND REBUILD, AND IF YOUR NETWORK TOPOLOGY IS RELATIVELY SIMPLE, THEN ANSIBLE IS GREAT

GitHub Repo: github.com/Levantine-1/ansible

WHAT DO YOU MEAN?!

Terraform is stateful and declarative. If I modify the IAC to create a resource, terraform creates the resource and keeps track of it. If I delete the resource from the IAC, terraform will also delete the actual resource.

Ansible can be declarative (flawed declarative in my opinion), but it's not stateful. So say I add a user or package today. If I want to remove the package or user later, I have to add a step to remove the package or user in the playbook.

So that's what I meant by if your operational paradigm is to burn down the environment and rebuild it, Ansible is great! And having a simple network topology really helps with not having to keep track of a bunch of SSH Jumps.

If your environment must be persistent and have weird hickups with SSH, might I recommend Declarative and Stateful SaltStack? In any case, Ansible is winning the config management war, so it's the tool I'll use and recommend.
